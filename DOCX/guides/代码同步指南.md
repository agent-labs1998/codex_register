# 代码同步指南

> 本地代码 → GitHub → 服务器

---

## 一、提交到 GitHub

```bash
cd A:/Github/codex_register

# 1. 查看改了哪些文件
git status

# 2. 暂存修改的文件（不要 git add .，逐个添加）
git add src/xxx.ts src/yyy.ts

# 3. 提交
git commit -m "fix: 简短描述改了什么"

# 4. 推送到 GitHub
git push origin master
```

**注意：**
- `config.json` 在 `.gitignore` 里，不会被提交
- `codex_register_V1/`、`v1-1/`、`.playwright-cli/` 也不会被提交
- 不要用 `git add .`，避免提交敏感文件（tokens.txt、密码等）

---

## 二、同步到服务器

```bash
# 1. 上传修改的源文件到服务器
scp src/xxx.ts src/yyy.ts openai:/root/github/codex/src/

# 2. 服务器上重新构建
ssh openai "cd /root/github/codex && npm run build 2>&1 && sed -i 's/require(\"sqlite\")/require(\"node:sqlite\")/' bundle/index.cjs && echo 'done'"
```

**注意：**
- 服务器用 `tsx`（dev 模式）直接读源码，不构建也能跑
- 用 `npm run start`（构建模式）才需要重新构建
- `sed -i` 命令是修复 `node:sqlite` 模块路径（服务器 Node.js v22 特有）

---

## 三、一键同步（本地 → GitHub + 服务器）

```bash
# 提交 + 推送 + 同步服务器 + 构建
cd A:/Github/codex_register && \
git add src/xxx.ts && \
git commit -m "fix: 描述" && \
git push origin master && \
scp src/xxx.ts openai:/root/github/codex/src/ && \
ssh openai "cd /root/github/codex && npm run build 2>&1 && sed -i 's/require(\"sqlite\")/require(\"node:sqlite\")/' bundle/index.cjs && echo 'done'"
```

---

## 四、同步 config.json

config.json 不走 GitHub，手动同步：

```bash
# 本地 → 服务器
scp config.json openai:/root/github/codex/config.json

# 服务器 → 本地
scp openai:/root/github/codex/config.json config.json
```

---

## 五、验证代码一致性

```bash
# 对比本地和服务器的源文件 MD5
for f in src/xxx.ts src/yyy.ts; do
  local=$(md5sum A:/Github/codex_register/$f | awk '{print $1}')
  remote=$(ssh openai "md5sum /root/github/codex/$f | awk '{print \$1}'")
  if [ "$local" = "$remote" ]; then echo "✅ $f"; else echo "❌ $f"; fi
done
```

## 六、服务器上的部署路径

| 路径 | 用途 |
|------|------|
| `/root/github/codex/` | 主项目代码 |
| `/root/github/codex/src/` | TypeScript 源码 |
| `/root/github/codex/bundle/` | 构建产物 |
| `/root/github/codex/config.json` | 配置文件 |
| `/root/github/codex/data/` | SQLite 数据库 |
| `/root/github/codex/logs/` | 运行日志 |
| `/root/github/db-viewer/` | DB Viewer Web 服务 |

## 七、SSH 连接方式

```bash
# 直接登录服务器
ssh openai

# 执行单条命令
ssh openai "命令"

# 上传文件
scp 本地文件 openai:/目标路径/
```

## 八、DB Viewer 服务管理

```bash
# 重启 DB Viewer
ssh openai "systemctl restart db-viewer"

# 查看状态
ssh openai "systemctl status db-viewer"

# 访问地址
http://服务器IP:8002
```
