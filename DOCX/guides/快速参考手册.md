# Codex-Register 快速参考手册

## 一、快速开始

### 1. 安装和配置
```bash
# 克隆项目
cd A:\Github\codex_register

# 安装依赖
npm install

# 复制并编辑配置
cp config.example.json config.json
# 编辑 config.json 填入你的配置
```

### 2. 基本运行

#### CPA 模式（最常用）
```bash
npm run dev -- --codex-cpa --email your@outlook.com
```

#### 手机号注册模式
```bash
npm run dev -- --phone
```

#### 邮箱注册模式
```bash
npm run dev -- --sign --at --email your@outlook.com
```

#### 仅获取 Token
```bash
npm run dev -- --auth --email your@outlook.com --at
```

## 二、命令行参数速查

### 核心参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--codex-cpa` | 启用 CPA 模式 | `--codex-cpa` |
| `--phone` | 手机号注册模式 | `--phone` |
| `--sign` | 直接 OAuth 注册 | `--sign` |
| `--at` | 获取 ChatGPT token | `--at` |
| `--auth` | 仅授权模式 | `--auth` |
| `--email` | 指定邮箱 | `--email user@hotmail.com` |
| `--password` | 指定密码 | `--password mypass123` |
| `--phone` | 指定手机号（复用） | `--phone +573001234567` |
| `--otp` | 手动输入 OTP | `--otp` |
| `--n` | 执行轮数 | `--n 10` |

### 输出参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--token-out` | Token 输出文件 | `--token-out tokens.txt` |
| `--gp-token-out` | GP Token 输出文件 | `--gp-token-out gp_tokens.txt` |
| `--callback-out` | Callback URL 输出 | `--callback-out callback.json` |

### CPA 参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--cpa-base` | CPA 基础 URL | `--cpa-base https://cpa.example.com` |
| `--cpa-key` | CPA 管理密钥 | `--cpa-key your_key` |
| `--bind-email` | 绑定邮箱 | `--bind-email bind@hotmail.com` |

### GP Plus 参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--gp-plus` | 启用 GP Plus 订阅 | `--gp-plus` |
| `--gp-script` | GP Python 脚本路径 | `--gp-script path/to/script.py` |
| `--gp-cwd` | GP 脚本工作目录 | `--gp-cwd path/to/dir` |

### 探测参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--probe-trial-jp` | 试用探测代理 | `--probe-trial-jp socks5://...` |
| `--profile-geo-check` | 仅检查注册代理出口国家和 locale | `--profile-geo-check` |

## 三、常见使用场景

### 场景 1：CPA 模式批量注册

```bash
# 单个注册
npm run dev -- --codex-cpa \
  --cpa-base https://cpa.example.com \
  --cpa-key your_key \
  --token-out tokens.txt

# 批量注册
npm run batch
```

### 场景 2：手机号注册并获取 Token

```bash
npm run dev -- --phone \
  --token-out phone_tokens.txt \
  --gp-token-out gp_tokens.txt
```

### 场景 3：邮箱注册并保存 Token

```bash
npm run dev -- --sign --at \
  --email newuser@hotmail.com \
  --token-out email_tokens.txt
```

### 场景 4：已有账户获取 Token

```bash
npm run dev -- --auth \
  --email existing@hotmail.com \
  --at \
  --gp-token-out existing_tokens.txt
```

### 场景 5：CPA + GP Plus 订阅

```bash
npm run dev -- --codex-cpa \
  --gp-plus \
  --cpa-base https://cpa.example.com \
  --cpa-key your_key \
  --token-out tokens.txt
```

### 场景 6：复用已有手机号

```bash
npm run dev -- --phone \
  --phone-existing +573001234567 \
  --token-out tokens.txt
```

## 五、日志标签说明

| 标签 | 含义 | 处理方式 |
|------|------|---------|
| `[✅️]` | 操作成功 | 无需处理 |
| `[❌️]` | 操作失败 | 检查错误信息 |
| `[⚠️]` | 警告 | 检查配置或继续 |
| `[POOL-RESULT]` | 池结果 | 记录用于统计 |
| `[codex-cpa]` | CPA 模式日志 | 关注状态 |
| `[phone-signup]` | 手机注册日志 | 关注状态 |
| `[profile-geo]` | 注册资料 locale 检测日志 | 观察出口国家/生成 locale |
| `SMS_RELEASE_DEADLINE` | 短信等待超时释放日志 | 说明号码因超时被释放 |
| `[巡视释放]` | 号码主动取消日志 | 说明程序已尝试立即释放供应商号码 |
| `[hotmail 卡密消费]` | 消费日志 | 记录用于审计 |

## 七、故障排查速查

### 问题 1：邮箱验证超时

**症状**：等待 OTP 超时
**检查**：
```bash
# 1. 检查 IMAP 配置
# 2. 检查卡密池是否有可用邮箱
cat hotmail/tokens.txt

# 3. 手动测试 IMAP 连接
```

**解决**：
- 更新 hotmail tokens
- 检查 IMAP 凭据
- 增加 `heroSMSPollIntervalMs`

### 问题 2：SMS 验证失败

**症状**：无法获取 OTP
**检查**：
```bash
# 1. 检查 API 密钥
grep heroSMSApiKey config.json

# 2. 检查余额（访问 Hero SMS 仪表板）

# 3. 检查国家代码（应为 33 = 哥伦比亚）
```

**解决**：
- 更新 API 密钥
- 充值账户
- 降低 `heroSMSMaxPrice`

### 问题 3：代理连接失败

**症状**：连接超时或拒绝
**检查**：
```bash
# 1. 验证代理格式
# socks5://user:pass@host:port

# 2. 测试代理连接
curl --socks5 user:pass@host:port https://httpbin.org/ip
```

**解决**：
- 检查代理凭据
- 更换代理服务器
- 检查防火墙设置

### 问题 4：Token 获取失败

**症状**：无法获取 ChatGPT access_token
**检查**：
```bash
# 1. 检查账户状态
npm run check

# 2. 检查 CPA 配置
npm run check:cpa
```

**解决**：
- 检查 OpenAI 账户状态
- 验证 CPA 配置
- 重新登录账户

### 问题 5：Cloudflare 拦截

**症状**：HTTP 403 错误
**检查**：
```bash
# 1. 更换代理
# 2. 清除 cookie
# 3. 更换设备指纹
```

**解决**：
- 使用不同的代理
- 增加请求间隔
- 更新设备配置

## 十一、常见问题解答

### Q1: 如何更换邮箱提供商？

**A**: 修改 `config.json` 中的 `provider` 字段：
```json
{
  "provider": "gmail"  // 或 "cloudflare", "gptmail"
}
```

### Q2: 如何使用不同的代理？

**A**: 修改 `config.json` 中的 `defaultProxyUrl`：
```json
{
  "defaultProxyUrl": "socks5://user:pass@new-host:port"
}
```

### Q3: 如何批量处理多个账户？

**A**: 使用批量注册命令：
```bash
npm run batch
```

或在脚本中循环调用：
```bash
for i in {1..10}; do
  npm run dev -- --codex-cpa --token-out "tokens_$i.txt"
done
```

### Q4: 如何验证注册是否成功？

**A**: 检查以下内容：
1. 日志中是否有 `[✅️]` 标记
2. Token 文件是否有新 token
3. 运行 `npm run check` 检查配额

### Q5: 如何恢复失败的注册？

**A**: 失败的号码会被自动标记，程序会自动重试。手动重试：
```bash
npm run dev -- --codex-cpa --phone-existing +57xxxxxxxxx
```

## 十二、进阶用法

### 自定义脚本集成

```bash
#!/bin/bash
# register_and_upload.sh

# 注册并获取 token
TOKEN=$(npm run dev -- --codex-cpa 2>&1 | grep "access_token" | cut -d' ' -f2)

# 上传 token 到服务器
curl -X POST https://api.example.com/tokens \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"token": "'$TOKEN'"}'
```

```bash
# 每小时运行一次

```

```dockerfile
FROM node:18

```
