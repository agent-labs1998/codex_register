## 二、当前项目已经完成了什么

### 1. CPA 注册流程可跑通
当前项目已经可以完成：

- coroabet 临时邮箱创建
- HeroSMS 取号
- OpenAI 手机注册
- SMS 验证码接收
- CPA OAuth
- 绑定邮箱
- 邮箱验证码接收
- CPA 入库
- token 写入文件

### 2. 已去掉部分硬编码
CPA 模式的邮箱绑定已经从原来固定 Hotmail 改为使用配置的邮箱 provider。

当前默认邮箱 provider 是：
- `coroabet`

### 3. 已支持自动注册资料 locale
现在已经支持根据注册代理出口 IP 自动选择注册资料的国籍风格。

配置项：
- `profileLocale`
- `profileAgeMin`
- `profileAgeMax`
- `profileIpLookupUrl`
- `profileIpLookupTimeoutMs`
- `profileLocaleByCountry`

关键模块：
- `src/profile-geo.ts`
- `src/profile-generator.ts`

当前逻辑：
- `profileLocale=auto` 时根据注册 IP 国家自动选择 faker locale
- 查询失败时回退到 `en_US`

### 4. 已实现巡视释放模式
当前已经实现两个层级：

#### A. 当前等待 deadline
在 CPA 等待短信时，如果超时未收到验证码：
- 主动尝试取消供应商号码
- 抛出超时错误
- 进入失败/重建逻辑

#### B. 独立后台巡视器
`src/sms/hero-patrol.ts` 已经实现：
- 程序运行期间持续轮询 active activations
- 发现超过阈值仍未收到验证码的号码，主动取消

当前目标是避免号码长时间挂死、浪费余额。

### 5. 已支持失败后从零重建 worker
当前 CPA 模式在短信失败/超时后，不再复用旧 worker：
- 重新生成设备指纹
- 重新准备邮箱
- 重新取号
- 重新发起注册

### 6. 邮箱前缀已配置化
`coroabet` provider 的邮箱前缀现在通过配置控制：

```json
"coroabetEnablePrefix": false
```

默认不加 `tmp` 前缀。

## 四、当前架构的关键点

### 注册资料生成
```text
注册代理出口 IP
  -> profile-geo.ts 检测国家
  -> profile-generator.ts 选 faker locale
  -> 生成 name / birthdate
```

### 邮箱
```text
coroabet provider
  -> POST /admin/new_address 创建邮箱
  -> GET /admin/mails 读取原始邮件
  -> 提取验证码
```

### SMS
```text
HeroSMS 取号
  -> 发短信
  -> 轮询验证码
  -> 超时则巡视释放
```

### 巡视器
```text
hero-patrol.ts
  -> 后台持续轮询
  -> 发现超时号码
  -> setStatus=8 尝试取消
```

## 六、新 agent 接手时不要做的事

### 不要硬编码密钥
所有敏感信息必须来自：
- `config.json`
- 环境变量
- CLI 参数

### 不要直接改旧脚本式写法而不留迁移路径
当前主入口仍是 `src/index.ts`，改造时注意：
- 保留原有单次运行能力
- 新 workflow 能力尽量增量添加

### 不要忽略释放逻辑
任何涉及号码/worker 的改动，都要同时考虑：
- 超时释放
- 失败释放
- 余额回笼
- 状态可追溯
