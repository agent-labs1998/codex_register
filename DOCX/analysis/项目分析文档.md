# Codex-Register 项目文档
## 项目概述
**项目名称**：codex-register
**版本**：1.0.7
**类型**：Node.js/TypeScript 自动化工具
**用途**：ChatGPT/OpenAI 账户自动化注册和管理

## 项目结构统计
- **总文件数**：37 个源代码文件
- **代码行数**：~8000+ 行 TypeScript 代码
- **模块数**：5 个核心模块（mail、sms、auth、proxy、automation）
- **支持邮箱**：Hotmail、Gmail、Cloudflare、GPTMail
- **支持代理**：SOCKS5、HTTP 代理

---

## 核心模块分析

### 1. 入口模块 (src/index.ts)
**文件大小**：~800 行
**功能**：程序主入口，处理命令行参数和流程编排
**支持的运行模式**：
- `--codex-cpa`：CPA 模式（Codex OAuth + CPA 入库）
- `--phone`：手机号注册模式
- `--sign`：直接 OAuth 签名模式
- `--at`：获取 ChatGPT accessToken 模式
- `--auth`：仅授权模式
- 自动轮询模式（无参数）

**关键特性**：
- 支持多种注册模式切换
- 支持批量注册（batch mode）
- 支持 token 管理和持久化
- 集成试用探测功能
- 支持 GP Plus 订阅链路
- 支持按注册代理 IP 自动选择注册资料 locale
- 支持 coroabet 临时邮箱绑定
- 支持短信等待巡视释放模式（超时未收到验证码立即释放号码并主动尝试取消供应商挂单）
- 号码/邮箱/worker 失败后支持从零重建，不复用脏状态

---

### 2. OpenAI 客户端模块 (src/openai.ts)
**功能**：封装与 OpenAI/Auth0 服务的交互
**核心类**：`OpenAIClient`
**主要方法**：
- `authRegisterHTTP()` - 邮箱注册
- `authLoginHTTP()` - 邮箱登录
- `authPhoneSignupHTTP()` - 手机号注册
- `authLoginChatGPTWeb()` - ChatGPT Web 登录
- `authRegisterAndAuthorizeHTTP()` - 注册并授权
- `getChatGPTAccessToken()` - 获取 ChatGPT 访问 token
- `saveChatGPTAccessToken()` - 保存 token 到文件

**技术实现**：
- 使用 Playwright 进行浏览器自动化
- 支持 Cloudflare 绕过
- 实现 Auth0 PKCE 认证流程
- 处理多步骤验证（邮箱、手机、TOTP）

---

### 3. 邮箱模块 (src/mail/)
**支持的邮箱类型**：

#### 3.1 Hotmail 邮箱 (src/mail/hotmail.ts)
- IMAP 连接读取验证邮件
- 自动 refresh token 管理
- 支持卡密池（tokens.txt）
- 自动验证码提取

#### 3.2 Gmail 邮箱 (src/mail/gmail.ts)
- IMAP over TLS
- 应用专用密码支持
- 邮件过滤和搜索

#### 3.3 Cloudflare 邮箱 (src/mail/cloudflare.ts)
- Cloudflare Email Routing
- IMAP 读取转发邮件
- 临时邮箱支持

#### 3.4 GPTMail 邮箱 (src/mail/gptmail.ts)
- 自建邮箱服务
- API + IMAP 双重访问
- 自动域名选择

#### 3.5 代理邮箱 (src/mail/proxiedmail.ts)
- 通过代理访问邮箱
- 绕过地区限制

#### 3.6 coroabet 临时邮箱 (src/mail/coroabet.ts)
- 使用自建 Cloudflare Worker 临时邮箱服务
- 通过 admin API 创建地址
- 通过 admin API 读取原始邮件并提取验证码
- 当前用于 CPA 模式的 `add-email` 绑定流程

---

### 4. SMS 短信模块 (src/sms/)
**架构**：
- `provider.ts` - SMS 提供商接口定义
- `heroSMS.ts` - Hero SMS 服务实现
- `activation-broker.ts` - 号码租赁和管理
- `index.ts` - 工厂方法

**支持的服务**：
- Hero SMS（哥伦比亚号码）
- 自动号码租赁
- 验证码轮询
- 失败重试机制

**配置项**：
```json
{
  "heroSMSApiKey": "API 密钥",
  "heroSMSCountry": 33,
  "heroSMSMaxPrice": 0.08,
  "heroSMSPriceTiers": [0.045, 0.05, 0.055, 0.06, 0.065, 0.07, 0.075, 0.08],
  "heroSMSPollAttempts": 15,
  "heroSMSPollIntervalMs": 3000
}
```

### 5. CPA 集成模块 (src/cpa-codex.ts)
**功能**：CliproxyAPI (CPA) 集成
**主要方法**：
- `requestCodexAuthUrl()` - 获取 OAuth 授权 URL
- `submitOAuthCallback()` - 提交 OAuth 回调
- `listAuthFiles()` - 列出认证文件
- `downloadAuthFile()` - 下载认证文件

**用途**：
- Codex CLI OAuth 流程
- Token 入库管理
- 批量账户管理

---

### 6. 设备配置模块 (src/device-profile.ts)
**功能**：生成随机设备指纹
**包含信息**：
- User-Agent
- 屏幕分辨率
- 语言和时区
- 浏览器特性

**目的**：避免被检测为自动化工具

---

### 6.1 代理调度模块 (src/proxy-dispatcher.ts)
**功能**：统一抽取 HTTP/SOCKS 代理调度能力
**目的**：供 OpenAI 注册、IP 地理检测等模块复用，避免重复实现

---

### 6.2 注册 IP 地理检测模块 (src/profile-geo.ts)
**功能**：通过注册代理出口 IP 查询当前国家
**用途**：
- 自动判断注册资料应使用哪种 locale
- 结果按代理地址和查询地址缓存，减少重复请求

---

### 6.3 注册资料生成模块 (src/profile-generator.ts)
**功能**：根据配置或注册代理出口 IP 生成 OpenAI 注册资料
**当前能力**：
- `profileLocale = auto` 时自动选择 locale
- `profileLocale` 显式指定时强制使用该 locale
- 仅生成 `name` 和 `birthdate`
- 不生成邮箱、手机号、地址等信息

**说明**：
- 邮箱仍使用 coroabet / 其他真实邮箱 provider
- 手机号仍使用 HeroSMS

## 运行流程

### 流程 1：CPA 模式 (--codex-cpa)
```
1. 读取配置和参数
2. 准备 Hotmail 邮箱（卡密池）
3. 获取 SMS 号码（hero-sms）
4. 手机号注册（phone signup）
5. 请求 CPA OAuth URL
6. 走 OAuth 登录流程
7. 提交 OAuth 回调给 CPA
8. 从 CPA 拉取 auth 文件
9. 获取 access_token
10. 试用探测（可选）
11. 写入 token 文件
12. 消费 hotmail 卡密
13. 启动 GP Plus 订阅（可选）
```

### 流程 2：手机号模式 (--phone)
```
1. 从 hero-sms 获取号码
2. 手机号注册（phone signup）
3. 建立 ChatGPT Web session
4. 获取 ChatGPT access_token
5. 保存 token 到文件
```

### 流程 3：邮箱注册模式 (--sign --at)
```
1. 生成随机邮箱（或使用指定邮箱）
2. 邮箱注册（Auth0 流程）
3. 读取验证邮件获取 OTP
4. 完成注册
5. 获取 ChatGPT access_token
6. 保存 token
```

## 故障排查

### 常见问题
1. **邮箱验证超时** - 检查 IMAP 配置和卡密池
2. **SMS 验证失败** - 检查 hero-sms API 密钥和余额
3. **代理连接失败** - 验证 SOCKS5 配置
4. **Token 获取失败** - 检查 OpenAI 账户状态
5. **Cloudflare 拦截** - 更换代理或设备指纹

### 错误码
- `exit 0` - 成功
- `exit 1` - 失败
- `exit 2` - 无试用资格
