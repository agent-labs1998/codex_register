# 任务书：邮箱去重管理 + 孤儿账号存储

> 优先级：高 | 预计工作量：1 小时 | 审核人：CTO

---

## 一、任务背景

当前注册流程存在两个缺陷：

### 缺陷 1：邮箱没有去重管理
Hotmail 邮箱池（`hotmail/tokens.txt`）有 17 个账号，程序从头到尾循环使用，没有标记"已用"。用完一轮后重复使用已注册过 OpenAI 的邮箱，导致 `email_already_in_use` 错误。

### 缺陷 2：孤儿账号没有存储
当注册走到最后一步（邮箱绑定）失败时，OpenAI 账号已经创建了（手机号已注册），但 CPA 没有收到 token，数据库里也没有记录。这个账号就"丢失"了。

```
取号 ✅ → 注册 OpenAI ✅ → 收到短信 ✅ → 绑定邮箱 ❌
                                                 ↓
                                          手机号已在 OpenAI 注册成功
                                          但没有 CPA token
                                          数据库无记录 ← 丢失
```

---

## 二、目标

### 目标 1：Hotmail 邮箱去重
- 用过的邮箱标记为 `used`，不再分配
- 每次取邮箱时，从未使用的列表中取第一个
- 所有邮箱用完时，给出明确的错误提示

### 目标 2：孤儿账号存储
- 邮箱绑定失败时，把手机号、邮箱、失败原因存入数据库
- 方便后续手动补救（手动绑定邮箱、手动导入 CPA）

## 四、具体改动要求

### 任务 A：Hotmail 邮箱去重

#### A1：数据库新增 `hotmail_accounts` 表

在 `src/local-db.ts` 中新增表：

```sql
CREATE TABLE IF NOT EXISTS hotmail_accounts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  email TEXT NOT NULL UNIQUE,
  password TEXT NOT NULL,
  client_id TEXT,
  refresh_token TEXT,
  status TEXT NOT NULL DEFAULT 'unused',  -- unused / used / failed
  used_at TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX IF NOT EXISTS idx_hotmail_status ON hotmail_accounts(status);
```

#### A2：启动时导入邮箱

在程序启动时（可以放在 `local-db.ts` 的初始化中，或 `index.ts` 启动时），读取 `hotmail/tokens.txt`，把未导入的账号插入 `hotmail_accounts` 表（`INSERT OR IGNORE` 避免重复）。

tokens.txt 格式：
```
email----password----client_id----refresh_token
```

每行 4 个字段，用 `----` 分隔。

#### A3：修改 Hotmail 邮箱获取逻辑

修改 `src/mail/hotmail.ts` 的 `getEmailAddress()` 方法：
1. 从 `hotmail_accounts` 表查询 `status='unused'` 的第一条记录
2. 标记为 `status='used'`，设置 `used_at = datetime('now')`
3. 返回该邮箱地址
4. 如果没有 `unused` 的邮箱，抛出明确错误：`"Hotmail 邮箱池已用完，请补充新账号"`

#### A4：注册失败时回滚邮箱状态

在 `cpa-registration.ts` 和 `concurrent-registration.ts` 中，当邮箱绑定失败（`email_already_in_use`）时：
- 如果是 `email_already_in_use` 错误 → 邮箱标记为 `status='failed'`
- 如果是其他错误（网络超时等）→ 邮箱回滚为 `status='unused'`（可重试）

### 任务 B：孤儿账号存储

#### B1：数据库新增 `orphaned_accounts` 表

在 `src/local-db.ts` 中新增表：

```sql
CREATE TABLE IF NOT EXISTS orphaned_accounts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  phone TEXT NOT NULL,
  email TEXT NOT NULL,
  password TEXT NOT NULL,
  activation_id TEXT,
  error_type TEXT NOT NULL,          -- email_already_in_use / email_otp_failed / cpa_callback_failed / other
  error_message TEXT,
  sms_code TEXT,                     -- 收到的短信验证码（如果有的话）
  openai_registered INTEGER DEFAULT 1,  -- OpenAI 是否已注册成功
  resolved INTEGER DEFAULT 0,        -- 是否已手动解决
  resolved_at TEXT,
  resolved_note TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX IF NOT EXISTS idx_orphaned_resolved ON orphaned_accounts(resolved);
CREATE INDEX IF NOT EXISTS idx_orphaned_phone ON orphaned_accounts(phone);
```

字段说明：
- `phone`：注册的手机号（OpenAI 已接受）
- `email`：绑定失败的邮箱
- `password`：注册密码
- `activation_id`：HeroSMS 激活 ID（可选，方便追溯）
- `error_type`：错误分类（方便筛选）
- `error_message`：完整错误信息
- `sms_code`：收到的验证码（如果有的话，方便手动操作）
- `openai_registered`：1=OpenAI 账号已创建，0=未创建
- `resolved`：0=未解决，1=已手动解决
- `resolved_at`：解决时间
- `resolved_note`：手动备注

## 六、验收标准

1. `npm run build` 构建成功
2. Hotmail 邮箱用过一次后不会再被分配（查 `hotmail_accounts` 表确认 status='used'）
3. 所有 Hotmail 邮箱用完后，程序报明确错误："Hotmail 邮箱池已用完"
4. `email_already_in_use` 错误时，孤儿账号存入 `orphaned_accounts` 表
5. `--db-list-orphans` 能正确显示未解决的孤儿账号
6. `--db-resolve-orphan` 能正确标记为已解决
7. coroabet 模式不受影响（coroabet 每次生成新邮箱，不需要去重）

---

## 七、注意事项

1. **不要改变 coroabet 的逻辑** — coroabet 每次生成新邮箱，天然不重复，不需要去重
2. **tokens.txt 格式** — `email----password----client_id----refresh_token`，用 `----` 分隔
3. **孤儿账号也要记 sms_code** — 有的话存进去，方便后续手动操作
4. **并发模式也要处理** — `concurrent-registration.ts` 中同样要存储孤儿账号
5. **数据库只用 SQLite** — 不要引入新依赖
