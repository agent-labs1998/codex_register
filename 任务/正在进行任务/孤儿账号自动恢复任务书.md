## 一、任务背景

注册流程中，有些手机号已经在 OpenAI 上注册成功，但因为邮箱绑定失败（`email_already_in_use`、邮箱验证码收不到等），CPA 没有收到 token，变成了"孤儿账号"。

这些账号可以手动补救：用同一个手机号+密码登录 OpenAI，绑定一个新邮箱，完成 CPA 入库。

**目标：** 实现自动化恢复，程序从 `orphaned_accounts` 表读取未解决的记录，自动登录、绑新邮箱、入库。

## 三、核心逻辑

### 恢复流程（每个孤儿账号）

```
Step 1: 从 orphaned_accounts 表读一条 resolved=0 的记录
        → 得到 phone、password

Step 2: 创建新邮箱
        → 调用 mailbox.getEmailAddress()
        → 注意：不要用之前失败的那个邮箱！

Step 3: 用手机号+密码登录 OpenAI
        → 创建 OpenAIClient({ email: phone, password })
        → 调用 authLoginHTTP()

Step 4: OAuth 流程中会跳转到 /add-email
        → 提交新邮箱（sendAddEmail）
        → 收邮箱验证码（getEmailVerificationCode）
        → 提交验证码

Step 5: 拿到 callback URL
        → 提交给 CPA 入库（submitOAuthCallback）

Step 6: 入库成功
        → 更新 orphaned_accounts: resolved=1, resolved_at=now, resolved_note="自动恢复成功"
        → 保存到 accounts 表

Step 7: 入库失败
        → 更新 resolved_note 记录失败原因
        → 不标记 resolved，下次可以重试
```

创建一个独立的恢复模块：

```typescript
export async function recoverOrphans(options: {
  db: LocalDB;
  maxAttempts?: number;  // 最多恢复几条，默认 10
  cpaBase: string;
  cpaKey: string;
}): Promise<{ success: number; failed: number; skipped: number }>
```

核心逻辑：
1. 从 `orphaned_accounts` 查询 `resolved=0` 的记录
2. 逐条尝试恢复
3. 每条恢复流程复用现有的 `OpenAIClient` + `authLoginHTTP`
4. 恢复成功/失败都更新数据库

**注意：**
- 恢复时要创建**全新的邮箱**，不能用之前失败的那个
- 密码从孤儿记录中读取（`password` 字段）
- 如果邮箱池用完了（Hotmail 模式），提前终止并提示
- 每条恢复之间加随机延迟（10-30 秒），避免触发风控

const client = new OpenAIClient({
    email: orphan.phone,       // 用手机号作为 email（OpenAI 登录用）
    password: orphan.password,
    deviceProfile: generateRandomDeviceProfile(),
    manualMode: false,
    bindEmail: newEmail,       // 新生成的邮箱

const authResult = await submitOAuthCallback(cpaBase, cpaKey, result.callbackURL);

## 七、注意事项

1. **不要修改 `openai.ts` 的 authLoginHTTP 方法** — 直接复用，不要改它的逻辑
2. **邮箱一定要新生成** — 不能用孤儿记录里的旧邮箱
3. **密码从 orphan 记录读取** — 不要硬编码
4. **coroabet 和 Hotmail 都要支持** — 根据 config.json 的 provider 决定用哪个邮箱
5. **Hotmail 模式要检查邮箱池** — 如果没有可用邮箱，提前终止
6. **失败时不要标记 resolved** — 保留为未解决，方便重试
