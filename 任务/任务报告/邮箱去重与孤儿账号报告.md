## 一、任务概览

本次任务实现了两个关键功能：
1. **Hotmail 邮箱去重**：防止重复使用已注册的邮箱
2. **孤儿账号存储**：在邮箱绑定失败时保存账号信息，方便后续手动补救

## 二、新增数据库表

### 1. `hotmail_accounts` 表（Hotmail 邮箱管理）

**文件**：`src/local-db.ts`

**表结构**：
```sql
CREATE TABLE IF NOT EXISTS hotmail_accounts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  email TEXT NOT NULL UNIQUE,
  password TEXT NOT NULL,
  client_id TEXT,
  refresh_token TEXT,
  status TEXT NOT NULL DEFAULT 'unused',  -- unused / used / failed / retryable
  used_at TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX IF NOT EXISTS idx_hotmail_status ON hotmail_accounts(status);
```

**字段说明**：
- `email`：Hotmail 邮箱地址（唯一）
- `password`：邮箱密码
- `client_id`：OAuth client_id
- `refresh_token`：OAuth refresh_token
- `status`：状态（`unused` / `used` / `failed` / `retryable`）
  - `unused`：全新未使用的邮箱
  - `used`：已成功注册，永不再用
  - `failed`：邮箱已被占用（email_already_in_use），永不再用
  - `retryable`：网络超时等错误，可被其他手机号重试
- `used_at`：使用时间
- `created_at`：创建时间

### 2. `orphaned_accounts` 表（孤儿账号存储）

**文件**：`src/local-db.ts`

**表结构**：
```sql
CREATE TABLE IF NOT EXISTS orphaned_accounts (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  phone TEXT NOT NULL,
  email TEXT NOT NULL,
  password TEXT NOT NULL,
  activation_id TEXT,
  error_type TEXT NOT NULL,          -- email_already_in_use / email_otp_failed / cpa_callback_failed / other
  error_message TEXT,
  sms_code TEXT,                     -- 收到的短信验证码（如果有的话）
  openai_registered INTEGER DEFAULT 1,  -- OpenAI 是否已注册成功
  resolved INTEGER DEFAULT 0,        -- 是否已手动解决
  resolved_at TEXT,
  resolved_note TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
CREATE INDEX IF NOT EXISTS idx_orphaned_resolved ON orphaned_accounts(resolved);
CREATE INDEX IF NOT EXISTS idx_orphaned_phone ON orphaned_accounts(phone);
```

**字段说明**：
- `phone`：注册的手机号（OpenAI 已接受）
- `email`：绑定失败的邮箱
- `password`：注册密码
- `activation_id`：HeroSMS 激活 ID（可选，方便追溯）
- `error_type`：错误分类（`email_already_in_use` / `email_otp_failed` / `cpa_callback_failed` / `other`）
- `error_message`：完整错误信息
- `sms_code`：收到的验证码（方便手动操作）
- `openai_registered`：OpenAI 账号是否已创建（1=已创建，0=未创建）
- `resolved`：是否已手动解决（0=未解决，1=已解决）
- `resolved_at`：解决时间
- `resolved_note`：手动备注

## 四、核心逻辑：邮箱使用策略

### 策略概览
```
✅ 成功的邮箱 → 标记为 'used'，永不再用
✅ 邮箱已被占用（email_already_in_use）→ 标记为 'failed'，永不再用
✅ 网络超时等错误 → 标记为 'retryable'，可被其他手机号重试
✅ 所有邮箱都用完（没有 unused 和 retryable）→ 提示用户并结束任务
```

### 详细流程
```
1. 程序启动时，把所有邮箱导入数据库（hotmail_accounts 表）
   初始状态：status = 'unused'

2. 每次需要邮箱时，从数据库查询可用邮箱：
   ├─ 优先查询 status='unused'（全新邮箱）
   └─ 如果没有，查询 status='retryable'（可重试邮箱）

3. 获取到邮箱后，立即标记为 status='used'

4. 使用这个邮箱注册 OpenAI

5. 注册失败时，根据错误类型处理：
   ├─ email_already_in_use → 标记为 status='failed'（永久不再使用）
   └─ 其他错误（网络超时等）→ 标记为 status='retryable'（可被其他手机号重试）

6. 所有邮箱都用完（status 都不是 'unused' 或 'retryable'）
   → 抛出错误："Hotmail 邮箱池已用完，请补充新账号"
   → 结束任务
```

### 关键优势
1. **一个手机号配一个邮箱**：每个手机号都会分配一个独立的邮箱，避免重复
2. **立即标记**：获取邮箱后立即标记为已用，不会被其他手机号重复使用
3. **智能重试**：网络超时等错误会标记为可重试，允许其他手机号尝试
4. **明确错误提示**：所有邮箱用完时，明确提示用户并结束任务

## 八、注意事项

1. **不要改变 coroabet 的逻辑** — coroabet 每次生成新邮箱，天然不重复，不需要去重
2. **数据库向后兼容** — 新增的表使用 `IF NOT EXISTS`，不会影响现有数据库
3. **邮箱重试机制** — 网络超时等错误会标记为 `retryable`，允许被其他手机号重试
4. **孤儿账号包含 sms_code** — 有的话存进去，方便后续手动操作
