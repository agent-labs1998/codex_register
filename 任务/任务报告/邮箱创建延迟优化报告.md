# 邮箱创建延迟优化报告

> 执行日期：2026-06-20 | 状态：已完成 | 构建：通过

---

## 一、任务目标

将邮箱创建（coroabet 临时邮箱）从「取号后立即创建」改为「OpenAI 注册成功后再创建」，避免失败号码浪费邮箱资源。

---

## 二、改动文件清单

| 文件 | 改动类型 | 主要变更 |
|------|----------|----------|
| `src/cpa-registration.ts` | 接口 + 逻辑 | 接口字段改为可选，新增邮箱延迟创建逻辑 |
| `src/worker-scheduler.ts` | 删除 + 简化 | 删除 Step 2 邮箱准备块，task 对象去掉邮箱字段 |
| `src/concurrent-registration.ts` | 移动 | 邮箱准备从注册前移到注册成功后 |
| `src/index.ts` | 删除 + 简化 | 串行模式删除提前创建邮箱，传入 db |

## 三、详细改动对比

### 3.1 `src/cpa-registration.ts`

#### 接口变更

**改动前：**
```typescript
export interface RegistrationTask {
  // ...
  bindEmail: string;
  fetchAddEmailOtp: () => Promise<string>;
  // ...
}
```

**改动后：**
```typescript
export interface RegistrationTask {
  // ...
  bindEmail?: string;              // 改为可选
  fetchAddEmailOtp?: () => Promise<string>;  // 改为可选
  // ...
  db?: {                           // 新增：用于更新数据库记录
    updateWorkerSlot(workerId: string, data: Record<string, any>): void;
    updateAttempt(attemptId: number, data: Record<string, any>): void;
  };
}
```

#### 新增邮箱延迟创建逻辑（在 phone signup 成功后、CPA OAuth 前）

```typescript
// Step 2: 创建邮箱（延迟到注册成功后才创建，避免失败时浪费邮箱资源）
let bindEmail = task.bindEmail;
let fetchAddEmailOtp = task.fetchAddEmailOtp;

if (!bindEmail) {
  try {
    const mailbox = await import("./mailbox.js");
    bindEmail = await mailbox.getEmailAddress();
    fetchAddEmailOtp = async () => {
      const startedAt = Date.now();
      console.log(`[cpa-registration] ${workerId} 等待邮件 OTP for ${bindEmail}`);
      return await mailbox.getEmailVerificationCode(bindEmail, { minTimestampMs: startedAt });
    };
    console.log(`[CPA] 邮箱已创建: ${bindEmail}`);
    if (db) {
      db.updateWorkerSlot(workerId, { bind_email: bindEmail });
      db.updateAttempt(attemptId, { email: bindEmail });
    }
  } catch (error) {
    reportStatus("failed");
    return { status: "failed", /* ... */ error: `邮箱准备失败: ${...}` };
  }
}
```

### 3.3 `src/concurrent-registration.ts`

**改动：** 将 Step 1「准备邮箱」块从注册前移到注册成功后

**改动前：**
```
Step 0: IP 检测
Step 1: 准备邮箱          ← 在这里创建邮箱
Step 2: 注册 OpenAI + 等 SMS
Step 4: CPA OAuth
```

**改动后：**
```
Step 0: IP 检测
Step 1: 注册 OpenAI + 等 SMS
Step 2: 创建邮箱           ← 移到这里，注册成功后才创建
Step 4: CPA OAuth
```

Phone signup 失败的 error return 中 `email` 字段从 `bindEmail`（未定义）改为 `""`。

## 六、流程对比

### 改动前
```
取号 → 创建邮箱 → 注册 OpenAI → 等验证码 → 提交验证码 → CPA 入库
        ↑ 浪费！
        失败的号码也创建了邮箱
```

### 改动后
```
取号 → 注册 OpenAI → 等验证码 → 注册成功后才创建邮箱 → 提交验证码 → CPA 入库
                                      ↑ 按需创建
                                      只有注册成功才创建邮箱
```
